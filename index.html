~ No SQL

~ Not only SQL（对传统的关系型数据库的一个补充）

关系型数据库能够保证数据的完整性和（强）一致性。
完整性：
~ 实体完整性：每条记录都是独一无二的，没有冗余。（主键、唯一索引）
~ 参照完整性：子表的外键列必须参照父表的主键列。（外键约束）
~ 域完整性：录入的数据都是有效的。（数据类型、类型长度、非空约束、默认值约束、检查约束）

事务隔离级别：
read-uncommitted
read-committed —> Oracle
repeatable-read —> MySQL
serializable

事务的ACID特性：
~ Atomicity：原子性
~ Consistency：一致性（事务前后数据状态是一致的）
~ Isolation：隔离性（并发事务不能知道彼此的中间状态）
~ Duration：持久性（事务完成了需要持久化）

Redis —> Remote Dictionary Server
set k1 v1 ex 30
get k1

启动 redis：redis-server –port 端口号 –requirepass 密码 > redis.log &
链接 : redis-cli

~ 基本命令
- keys: 查看键
- exists：判断键是否存在
- del：删除指定的键
- expire：设置键过期时间（秒）
- ttl：查看键还有多少时间过期
- dbsize：数据库的大小（一共有多少组键值对）
- flushdb：清空数据库（删除当前数据库所有的键值对）
- shutdown：关闭Redis服务器
- save / bgsave：手动保存数据
